# PigeonDeck UI 风格文档

## 1. 风格目标

PigeonDeck 的 UI 目标不是“华丽”，而是“安静、温柔、雅致、可靠”。它应该有一点宣纸的气息，但不能走成古风，也不能像咖啡馆或文房陈设。整体感觉要克制、细腻、轻。

关键词：

- 宣纸感
- 温润留白
- 淡雅简朴
- 清简专业
- 高级但不堆装饰

## 2. 品牌气质

PigeonDeck 的品牌气质应像一个轻声说话、但判断清楚的工具。

- 品牌名应稳定、简洁、可记忆。
- logo 是一只抽象鸽子，不写实，不可爱化，不做复杂插画。
- 鸽子的处理方式应偏一笔线稿或极简轮廓，像符号，不像动物摄影。
- 品牌识别感来自轮廓、留白和比例，不来自花哨图形。

## 3. 色彩方向

整体色彩应偏暖，但不能黄得发旧，也不能变成咖啡色系。

- 主底色：暖白、米白、纸白一类的浅色。
- 主文字：深墨灰，清晰但不要发黑发硬。
- 点睛色：淡墨绿，作为激活、参考线、选中态和轻提示的主色。
- 辅助色：低饱和的浅灰、暖灰、纸色过渡。
- 禁止使用过多高饱和颜色。

色彩原则：

- 颜色只负责分层，不负责炫耀。
- 激活色要清楚，但不能喧宾夺主。
- 整体要有“看久了不累”的感觉。

## 4. 材质与层次

宣纸感不靠厚重纹理，而靠极轻的材质暗示。

- 纸纹理极弱，几乎不可见，细看才有。
- 浮层和工具盘主要靠细边框、轻阴影、微弱明暗差区分。
- 不做明显玻璃拟态，不做重渐变，不做大面积花纹。
- 卡片感要弱，层级感要清楚，但不要像一堆盒子。

## 5. 字体气质

字体要有一点现代宋感，但不能影响可读性。

- 标题可略带书卷气。
- 正文保持现代、清楚、干净。
- 整体不要太科技，也不要太书法。
- 语气和字形都要温柔，但信息表达必须准确。

## 6. 控件风格

控件应该像精致工具，而不是营销组件。

- 按钮默认图标优先，文字尽量短。
- 工具盘里不要把每个按钮都做成大卡片。
- 鼠标悬浮时才显示辅助文字或轻阴影。
- 常态下尽量弱化边界，激活或 hover 时再增强存在感。
- 图标线条细而有呼吸，不要粗、硬、满。

## 7. 空间与节奏

空间是这个项目的重要审美元素。

- 留白要足，但不是空洞。
- 节奏要中性均衡，不要密得像后台，也不要散得像海报。
- 空白区域要承担引导和呼吸的作用。
- 组件之间保持清晰分隔，但分隔方式要轻。

## 8. 动效气质

动效要慢而轻。

- 展开、收起、浮层出现都要柔和。
- 动效存在感要低，但能帮助用户理解状态变化。
- 不要弹跳，不要夸张，不要炫技。
- 不要让动效抢走内容和操作本身。

## 9. 关键 UI 要点

### 9.1 悬浮球与工具盘

- 悬浮球是品牌入口，像一个安静的标识。
- 展开后是紧凑 2x3 工具盘，不要做成长列卡片。
- 工具盘要像一个轻工具，不像一个大面板。
- 顶部横幅、拖拽把手、Undo/Redo 要明确，但不喧宾夺主。

### 9.2 批注与编辑面板

- 批注面板贴近目标元素。
- 面板内容要强调批注输入，再兼顾样式编辑。
- 高级样式区可以展开，但不能把主界面撑得臃肿。

### 9.3 参考线与批注标记

- 参考线和连线要清楚。
- 批注编号要轻盈，像小小的提示，不像强视觉标签。
- 选中态和移动态要明显，但只在需要时明显。

## 10. 硬性禁止项

以下内容在 UI 中明确禁止：

- emoji
- 竖线装饰
- 写实鸽子
- 明显装饰堆叠
- 重渐变
- 玻璃拟态
- 复杂纹理
- 花哨插画
- 卡片套卡片
- 过重边框
- 过多说明文字

## 11. 软性避免项

以下内容尽量避免：

- 过于古风
- 过于陈旧
- 过于泛黄
- 过于咖啡馆化
- 过于科技化
- 过于营销页化

## 12. 设计判断标准

如果某个视觉元素看起来“很努力”，大概率就不该要。  
如果某个元素只是为了让页面更像设计稿，那也大概率不该要。

PigeonDeck 的 UI 应该让人觉得：

- 轻
- 静
- 准
- 有气质
- 不吵
- 但很好用
