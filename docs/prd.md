# PigeonDeck 重开发 PRD

## 1. 产品目标

PigeonDeck 是一个面向网页验收、UI 修改反馈和 AI 编码交付的浏览器扩展。它以 MarkIt 为主要行为参考，从零使用 Vite + TypeScript + Manifest V3 重建一个更精简、可维护的版本；以 ClickDeck 作为移动组件、吸附参考线和移动任务 prompt 的局部参考。

第一版目标不是复刻 MarkIt 的全部复杂度，而是做到“体验接近 MarkIt，但工作流更统一”：保留标注、区域批注、文本/样式直接编辑、复制文本、复制图片、清空、设置、教程、popup 禁用、右键菜单和 file 页面支持；去掉 Dev Mode / Prototype Mode 双模式、高级输出模板、复杂 pin 样式合集、额外导出入口和营销型视觉效果。

## 2. 参考边界

### 2.1 MarkIt 参考内容

- 网页元素标注、区域框选、编号圆点与连线。
- 标注面板中的文本说明、元素内容修改、样式修改与即时预览。
- 当前视口截图与叠加批注后的图片复制。
- popup 中的全局禁用、站点禁用和 file 权限提示。
- 右键菜单快速标注入口。
- 首次使用教程与设置内手动重看教程。

### 2.2 ClickDeck 参考内容

- 移动组件时的 ghost/预览思路，但本项目采用移动真实元素预览。
- 同容器可见元素的吸附候选、边缘/中心线参考线、阈值匹配。
- 移动意图导出为 AI 任务时的规则：不要把视觉坐标直接写成硬编码 CSS，优先使用已有布局机制。

### 2.3 不参考或不进入 V1

- 不拆改 MarkIt 解包后的 `content.js`。
- 不保留 MarkIt 的 Dev Mode / Prototype Mode 双模式。
- 不保留 Markdown 导出、下载截图等额外导出入口；V1 只保留复制文本和复制图片。
- 不保留高级设置：多套 pin 样式、输出模板、详细度、多主题色等。
- 不默认对复制内容脱敏。

## 3. 用户与核心场景

目标用户是在网页上向 AI 编码工具表达修改意图的人，包括产品验收、前端 UI 调整、页面复核和原型评审场景。

核心场景：

1. 用户打开页面，点击 logo 悬浮球展开紧凑 2x3 工具盘。
2. 选择标注工具，点击页面元素，写一句或多句修改说明，并可直接修改文本或样式形成即时预览。
3. 长按拖拽框选页面区域，生成区域批注，填写一句修改说明。
4. 选择移动组件工具，单击选中组件，长按拖动到目标位置，默认吸附并显示参考线，按住 Alt 自由移动。
5. 点击复制文本，将所有标注、编辑、区域和移动预览整理成 Codex 可执行的任务清单。
6. 点击复制图片，只复制当前视口截图，并叠加编号、连线、区域框和移动预览。
7. 使用 Undo/Redo 回退或恢复操作；点击清空并二次确认后恢复当前页干净状态。

## 4. 功能需求

### 4.1 Logo 悬浮球与 2x3 工具盘

- 页面注入一个 logo 悬浮球，默认停靠在页面边缘。
- 点击 logo 悬浮球展开紧凑专业的 2x3 工具盘；再次点击 logo 收起工具盘。
- 工具盘默认向右下展开；靠近视口边缘时保持右下展开方向，但整体向视口内推，确保完整可见。
- 工具盘顶部小横幅包含：左侧 logo、中间明确的拖拽把手、右侧紧凑 Undo/Redo。
- 拖拽把手用于移动整体插件位置。
- 工具盘 2x3 顺序为：标注、移动组件、复制文本、复制图片、清空、设置。
- 工具按钮以图标为主，悬停时显示文字。
- 当前激活工具使用按钮高亮，并在工具盘外层显示轻量描边。
- Undo 和 Redo 是工具盘横幅中的常驻动作；Redo 是“反撤销”，不命名为返回。
- 复制文本/复制图片的成功或失败反馈在按钮内部短暂变化。
- 清空确认使用贴工具盘的小确认弹层。

### 4.2 元素选择与粒度切换

- 支持两级选择粒度：智能视觉组件块、精确 DOM 元素。
- 默认粒度可在设置中选择。
- 选中框附近提供紧凑的粒度切换按钮。
- 支持用户自定义快捷键快速切换粒度。
- 如果点击位置无法形成有效目标，显示工具栏轻提示并保持当前工具模式。

### 4.3 标注与直接编辑

- 标注模式下，点击元素后在目标附近显示批注浮层。
- 标注核心模型是 `target + instruction`。
- 批注浮层内集成样式/内容编辑，不拆成独立工具。
- 样式编辑第一版尽量保留 MarkIt 大部分常用项。
- 常驻编辑项包括：批注、文字、颜色、字重、字号、对齐方式、布局。
- 高级样式区使用左侧导航，并按使用频率从上到下排序；它应尽量覆盖 MarkIt 大部分样式项，包括尺寸、间距、背景、边框、圆角、阴影、透明度、媒体和调试/细节信息。
- 所有直接编辑都应即时预览，并进入撤销/重做历史。
- 复制文本时，样式修改以“人类指令 + CSS 属性前后值表”的形式输出。

### 4.4 区域批注

- 长按/拖拽框选区域后创建区域批注对象。
- 区域批注面板只要求用户填写一句修改说明。
- 区域批注不立即要求选择导出文本或图片；导出由工具栏的复制文本/复制图片按钮决定。
- 区域对象应记录视口位置、文档位置、附近元素摘要和用户说明。

### 4.5 移动组件

- 移动模式下，单击选中组件，长按拖动。
- 长按默认阈值为 300ms，可在设置中调整。
- 拖动时移动真实元素预览，预览保留到清空或撤销。
- 默认根据同容器可见元素计算吸附和参考线。
- 按住 Alt 时进入自由移动：不吸附、不显示参考线，松手后记录为 free move。
- 松手后自动生成移动任务，不弹出确认卡片。
- 同一个组件多次移动时，复制文本只输出“初始位置 -> 最终位置”；撤销/重做历史仍保留每一步移动。

### 4.6 复制文本

- 复制文本输出为任务清单格式，默认语言跟随界面语言。
- 输出包含页面上下文、全局编辑规则、操作列表和每个操作的定位信息。
- 操作类型至少包括：标注、区域批注、文本修改、样式修改、移动组件。
- 上下文保持精简：包含目标选择器、文本摘要、位置、必要 computed style，不默认包含完整 HTML/CSS。
- 移动任务必须说明 Source A、Target B、初始位置、最终位置、吸附参考线或 free move 状态。
- 输出应明确提示 AI：视觉坐标是定位线索，不应直接变成脆弱的 `top/left` 硬编码；优先使用现有布局、flex/grid、gap、margin、order 或局部 wrapper。

### 4.7 复制图片

- 复制图片只复制图片，不额外复制文本。
- 默认范围是当前视口。
- 图片叠加当前页面截图、批注编号、连线、区域框、移动预览。
- 是否在图片角落显示 URL/时间等元数据由设置开关控制。
- 复制失败时显示轻提示并保留所有状态，不自动改为复制文本。

### 4.8 清空

- 清空按钮第一次点击后，在工具盘旁出现小确认弹层。
- 小确认弹层包含确认清空和取消动作。
- 确认后清除当前页的标注、区域、直接编辑、移动预览、撤销/重做历史。
- 清空后编号从 1 重新开始。

### 4.9 设置

V1 设置保持最小必要：

- 界面语言。
- 默认选择粒度。
- 长按时长，默认 300ms。
- 撤销历史上限，默认 50 步。
- 快捷键设置，至少支持撤销、重做、粒度切换、退出当前工具。
- 工具栏位置重置。
- 自动清空策略。
- 图片元数据显示开关。
- 手动重新打开教程。

不进入 V1 的设置：复杂主题、多 pin 样式、输出模板、输出详细度、高级吸附策略。

### 4.10 教程

- 安装后打开说明页。
- 教程以安装说明页为主，不在页面内做完整教程浮层。
- 安装说明页按“快速上手流程 + 示例驱动 + 功能总览”组织。
- 设置中可手动打开说明页新标签。

### 4.11 Popup 与后台能力

- popup 支持全局禁用。
- popup 支持当前站点禁用和禁用列表管理。
- popup 在 file 权限未开启时提示用户前往扩展详情开启文件访问。
- 后台 service worker 支持右键菜单快速标注、截图消息转发和必要的权限/状态协调。

### 4.12 特殊页面

- 目标浏览器为 Chrome/Edge MV3。
- V1 完整支持 `file://` 页面。
- PDF 页面第一版提示不支持，不纳入核心验收。

## 5. 状态与数据规则

- 暂存生命周期按标签页会话实现。
- 同一 tab 刷新同一完整 URL 时自动恢复状态。
- 关闭 tab 后清理该 tab 会话状态。
- 页面键使用完整 URL。
- 如果刷新后找不到原移动组件，跳过移动预览并轻提示；移动任务记录保留，复制文本仍包含该意图。
- 如果刷新后找不到原编辑元素，跳过编辑预览并轻提示；编辑任务记录保留，复制文本仍包含该意图。
- 标注编号在当前会话中不因删除而重排；清空后重置。
- 撤销/重做覆盖全部编辑操作：标注、区域、直接编辑、移动预览、清空。

## 6. UI 预览清单

本阶段不生成 HTML UI mockup。以下内容只作为 PRD 内的低保真 UI 状态清单，后续还需要单独敲定 UI 风格、配色、图标、动效和最终布局。

### 6.1 页面内 UI 状态

- 悬浮球收起态。
- 悬浮球展开态，显示紧凑 2x3 工具盘。
- 工具盘顶部小横幅：左侧 logo、中间拖拽把手、右侧 Undo/Redo。
- 工具按钮图标为主，悬停显示文字。
- 当前激活工具按钮高亮，并在工具盘外层显示轻量描边。
- 标注工具激活态：hover 高亮、选中框、粒度切换按钮。
- 批注浮层：一句说明输入区、样式/内容编辑区、保存/删除动作。
- 区域框选态：拖拽矩形、区域批注浮层、区域编号。
- 移动工具激活态：组件选中框、真实元素预览、参考线、吸附点。
- Alt 自由移动态：隐藏参考线，显示 free move 状态提示。
- 复制文本成功/失败按钮内部反馈。
- 复制图片成功/失败按钮内部反馈。
- 贴工具盘的清空确认小弹层。
- 贴工具盘的设置面板。
- 安装说明页。

### 6.2 Popup UI 状态

- 当前站点状态。
- 当前站点禁用开关。
- 全局禁用开关。
- 禁用列表管理。
- file 权限提示。
- PDF 不支持提示入口或说明。

### 6.3 后续 UI 风格待定项

- 悬浮球品牌形态。
- 图标风格。
- 主色和状态色。
- 批注编号视觉。
- 连线和参考线视觉。
- 2x3 工具盘视觉密度。
- 设置面板信息密度。
- 安装说明页视觉风格。

- 不进入工程实现计划。

- 不把 `reference-projects/` 作为产品源码；后续实现前必须从 git 索引移出并加入忽略规则。

## 8. 验收标准

### 8.1 功能闭环

- 用户可以通过悬浮球完成标注、区域批注、直接编辑、移动组件、复制文本、复制图片、撤销/重做、清空和设置。
- popup 可以完成全局禁用、站点禁用、禁用列表管理和 file 权限提示。
- 右键菜单可以快速进入标注流程。
- file 页面可以正常使用核心功能。
- PDF 页面显示明确的不支持提示。

### 8.2 输出质量

- 复制文本能把所有操作组织成 Codex 可执行的任务清单。
- 同一组件多次移动时，输出只保留初始位置到最终位置。
- 样式修改包含用户意图和属性前后值。
- 复制图片只产生图片，不附带文本。
- 当前视口图片能看清批注编号、连线、区域框和移动预览。

### 8.3 状态恢复

- 同一 tab 刷新同一完整 URL 后，能自动恢复可定位的标注、编辑和移动预览。
- 找不到目标时不乱改页面，显示轻提示，并保留任务记录。
- 清空后页面状态、历史和编号全部重置。

### 8.4 Git 安全

- 后续实现前必须确保 `reference-projects/` 不进入新的提交。
- 如果 `reference-projects/` 已在索引中，执行阶段应仅移出索引并保留本地文件。

### 8.5 测试与验收环境

- 使用本地固定 HTML 夹具覆盖按钮、卡片、表单、图片、嵌套布局和可移动组件。
- 使用至少一个真实网站做手动冒烟测试。
- 覆盖 Chrome/Edge MV3 加载、popup、content script、background 截图消息、右键菜单和 file 页面权限路径。

- PRD 明确 UI 风格尚未敲定，后续需要单独进行 UI 风格讨论。
- 未生成 HTML UI mockup。
- 未修改源码。
- 未处理 git 索引。

产品 V1 结束标准：

- PRD 中的核心功能闭环全部可用。
- UI 风格经过用户确认。
- 复制文本和复制图片均满足验收标准。
- 参考项目文件不再进入提交。
