# PigeonDeck HTML 预览提示词

你是一位审美很好、但需要明确边界才能稳定发挥的 UI 设计 AI。请为 PigeonDeck 生成一个单页 HTML 预览草稿。这个草稿的目标是展示 UI 的气质、层级和状态，而不是完成最终视觉定稿。

## 任务目标

请生成一个兼顾审美和轻交互的单页 HTML 预览，重点帮助我判断：

- 整体气质是否温柔、雅致、克制
- 控件层级是否清楚
- 页面是否有高级感但不堆装饰
- 悬浮工具、批注面板、设置、清空确认和 popup 这些层级是否统一
- 在真实网页背景上是否仍然好看、安静、可用

## 你要理解的产品

PigeonDeck 是一个网页批注和 AI 交付工具。它的 UI 主入口不是传统侧边栏，也不是重型工作台。

收起时，页面上只能看到一个纯粹的品牌 logo 悬浮球。这个收起态不要出现文字、工具按钮、提示条或额外装饰。它应该像一个安静放在网页上的抽象鸽子标识。

点击这个 logo 后，才展开一个紧凑的 2x3 工具盘。展开后，原本的 logo 成为工具盘左上角的品牌锚点，工具盘顶部出现横幅，中间是拖拽把手，右侧是 Undo 和 Redo。页面上还会有批注面板、区域批注、移动组件、设置、清空确认和 popup 等状态。

你不需要写业务逻辑，但你要把这些状态完整展示出来，让我能看清它的气质和层级。

## 语言规则

预览里的可见界面文案默认全部使用简体中文，包括按钮、面板标题、提示、字段名、状态说明和示例内容。不要把界面做成英文产品稿。

唯一允许出现英文的位置是设置面板里的语言切换选项：`中文 / English`。这个控件用于表达未来支持中英文切换，但当前预览状态应选中中文。

## 程序里实际有什么

PigeonDeck 是一个浏览器扩展，不是网页应用本身。它覆盖在任意网页上，帮助用户把页面修改意见整理成 AI 可以执行的任务。

这个插件的核心内容是：

- 一个收起态只有 logo 的悬浮入口。
- 一个展开后的紧凑 2x3 工具盘。
- 六个主工具：标注、移动、复制文本、复制图片、清空、设置。
- 顶部历史操作：Undo、Redo。
- 页面元素 hover 高亮、选中框、区域框选、编号圆点、连线。
- 标注/编辑面板：写批注，也能直接修改目标元素的文字和样式。
- 移动组件模式：选中网页组件，拖动真实元素预览，显示吸附参考线。
- 复制文本：把所有标注、样式修改、文字修改、区域批注、移动任务整理成任务清单。
- 复制图片：只复制当前视口图片，图片里叠加编号、连线、区域框、移动预览，不额外附加文字。
- 清空确认：二次确认后清除当前页的批注、区域、编辑、移动预览和历史记录。
- 设置面板：只放必要设置，不做重型设置中心。
- 浏览器 popup：只负责扩展状态、全局禁用、站点禁用、file 权限提示和 PDF 不支持提示。

请把这些内容都体现在原型里。不要只画一个漂亮工具盘，也不要把它做成通用 UI kit。

## 风格要求

请把这个页面做成一种“宣纸感、温润留白、淡雅简朴、清简专业、高级但不堆装饰”的 UI。

你要表达的是感觉，不要把它做成重装饰设计稿。

整体应像：

- 安静
- 温柔
- 克制
- 有一点书卷气
- 现代但不冷
- 雅致但不旧

请控制得像一位很懂分寸的人在做工具，而不是在做海报。

## 必须保留的设计边界

### 绝对禁止

- 不要出现 emoji
- 不要出现竖线装饰
- 不要写实鸽子
- 不要显示任何域名
- 不要把界面文案写成英文，除语言切换里的 English 选项外都用简体中文
- 不要做明显装饰堆叠
- 不要做重渐变
- 不要做玻璃拟态
- 不要做复杂纹理
- 不要做花哨插画
- 不要做卡片套卡片
- 不要做过重边框
- 不要放太多说明文字

### 尽量避免

- 过于古风
- 过于陈旧
- 过于泛黄
- 过于咖啡馆化
- 过于科技化
- 过于营销页化

## 视觉方向

请把页面做成一种轻微的纸感，但纸感要非常克制，几乎不可见，只在细节里暗示。  
整体色调可以偏温润，不要冷得像后台，也不要黄得像旧纸。

品牌 logo 是一只抽象鸽子，只要在左上角作为品牌识别出现即可，不要把鸽子和功能做成很直白的拼图式结合。鸽子必须抽象，不要像真实照片或写实插画。

## 页面背景

请使用一个中性产品页作为背景，来测试插件在真实网页上的遮挡、层级、对比和存在感。

背景里可以有：

- 卡片
- 按钮
- 表单
- 图片
- 表格
- 嵌套布局
- 一些常见业务内容

背景不需要特别漂亮，但要足够像真实网页，方便测试插件覆盖层在复杂页面上的表现。

## 你要展示的层级

请在同一页里把这些状态都展示出来，尽量让它们彼此可见但不互相打架。

请特别注意：这是一个有明确层级关系的插件 UI，不是一组随意摆放的漂亮组件。你需要让观众一眼看出谁是入口、谁是展开后的工具盘、谁是贴近网页目标的上下文面板、谁是浏览器 popup。

### 1. 主入口

- logo 悬浮球收起态
- logo 悬浮球展开态
- 紧凑 2x3 工具盘
- 工具盘顶部横幅
- 顶部横幅左侧 logo
- 顶部横幅中间拖拽把手
- 顶部横幅右侧 Undo 和 Redo

主入口层级必须是：

1. 收起态只有 logo 悬浮球。
2. 点击 logo 后展开工具盘。
3. 展开后 logo 位于工具盘左上角。
4. 工具盘顶部横幅负责品牌、拖拽和历史操作。
5. 2x3 工具网格负责六个主功能。

不要把 logo 悬浮球和展开后的左上角 logo 画成两个同时独立存在的入口。它们是同一个入口在不同状态下的位置关系。

### 2. 工具盘按钮

- 标注
- 移动
- 复制文本
- 复制图片
- 清空
- 设置

请让按钮以图标为主，文字只在 hover 或需要强调时出现。  
按钮本身不要做成厚重卡片。  
更像精致的工具，不像大面板。

工具盘需要展示这些状态：

- 当前工具激活态，例如标注或移动被选中。
- 工具盘外层轻微描边，表示当前有工具激活。
- 复制文本成功/失败的按钮内部短反馈。
- 复制图片成功/失败的按钮内部短反馈。
- 清空按钮触发贴近工具盘的小确认弹层。
- 设置按钮打开贴近工具盘的设置面板。

### 3. 页面内状态

- 元素 hover 高亮
- 选中框
- 粒度切换按钮
- 批注浮层
- 区域批注
- 移动参考线
- 移动时的真实预览
- 自由移动状态
- 清空确认小弹层
- 设置面板

页面内状态的层级关系应是：

1. 最底层是中性产品页背景。
2. 其上是插件的悬浮入口和工具盘。
3. 再上是网页目标相关覆盖层，例如 hover 高亮、选中框、区域框、移动参考线。
4. 再上是上下文面板，例如批注/编辑面板、设置面板、清空确认小弹层。
5. 最轻的一层是按钮内部反馈和短提示。

不要把所有面板都做成同级大卡片。批注面板要和网页目标有关；设置面板要和工具盘有关；清空确认要和清空按钮有关。

### 4. 批注/编辑面板

批注/编辑面板是最重要的上下文面板。它应该贴近被选中的网页元素，而不是像普通弹窗一样居中。

面板里需要展示：

- 顶部：批注编号、目标元素摘要、关闭按钮。
- 主要输入区：批注说明 textarea，默认聚焦，提示用户写“希望 AI 怎么改”。
- 文字内容区：展示当前元素文字，可直接编辑。
- 常驻样式区：颜色、字重、字号、对齐方式、布局。
- 高级样式区：左侧导航 + 右侧当前分类控件。
- 底部动作：保存/更新、删除。
- 样式修改预览：可以用轻量 Before / After 形式展示属性变化。

常驻样式控件可以用这些形式：

- 颜色：色块/色板，不要只用文字。
- 字重：分段按钮或下拉。
- 字号：步进器、输入框或滑杆。
- 对齐：左对齐、居中、右对齐、两端对齐图标按钮。
- 布局：紧凑的 display / flex / grid / block 选择。

### 5. 高级样式区

高级样式区要像“可用的样式编辑器”，但不要臃肿。请用左侧导航把复杂度收住，右侧展示当前分类的控件。

左侧导航分类：

- 常用
- 文字
- 布局
- 间距
- 外观
- 媒体
- 细节

右侧控件示例：

- 常用：display、visibility、opacity、z-index、overflow。
- 文字：font-family、font-size、font-weight、line-height、letter-spacing、color、text-align、text-transform。
- 布局：position、width、height、min/max width、min/max height、flex direction、justify-content、align-items、grid columns。
- 间距：margin、padding、gap，可用四边联动/独立控制。
- 外观：background、border、border-radius、box-shadow、outline、filter。
- 媒体：object-fit、object-position、image size、image crop preview。
- 细节：元素 selector、DOM path、文本摘要、computed style 前后值。

请不要把所有高级控件全部摊开。原型里可以展示其中一两个分类的展开效果，但要让人看出整体结构完整。

### 6. 区域批注

区域批注是长按/拖拽框选页面区域后的状态，不是独立工具。

需要展示：

- 半透明区域框。
- 区域编号圆点。
- 从编号到区域或说明的细连线。
- 一个小型区域批注面板，只有一句说明输入、保存、删除。
- 区域批注要参与复制文本和复制图片，所以视觉上要和普通元素批注同一体系。

### 7. 移动组件模式

移动模式来自工具盘里的“移动”按钮。它需要表现得像真实网页组件被移动，而不是只画一个虚线框。

需要展示：

- 单击选中的组件选中框。
- 选中框附近的粒度切换按钮，例如元素 / 容器 两级。
- 长按拖动时，真实元素跟随移动的预览。
- 默认状态显示细参考线和吸附关系，例如边缘线、中心线、同容器对齐线。
- Alt 自由移动状态：不显示参考线，不吸附，只显示非常轻的 free move 状态提示。
- 已保存移动任务可以有轻量标记，表达“初始位置 -> 最终位置”，但不要画复杂轨迹。

### 8. 清空确认

清空确认是贴近工具盘的小弹层，不是全屏 modal。

需要展示：

- 一句简短确认文案。
- 确认清空按钮。
- 取消按钮。
- 它应该附着在清空按钮附近，层级高于工具盘但低于大型上下文面板。

### 9. 设置面板

设置面板从工具盘里的“设置”按钮打开，贴近工具盘。它是短表单，不是完整后台。

需要展示这些分区：

- 通用：界面语言切换（中文 / English，当前选中中文）、默认选择粒度、重置插件位置。
- 交互：长按时长、历史记录上限、快捷键。
- 输出：图片元数据显示开关。
- 会话：自动清空策略。

设置面板打开时，应显得页面选择和拖动被暂停。不要在设置里加入复杂主题、多套 pin 样式、多输出模板或输出详细度。

### 10. Popup 状态

- 当前站点状态
- 全局禁用
- 当前站点禁用
- file 权限提示
- PDF 不支持提示

Popup 是浏览器扩展图标点开后的小窗口，不属于网页内悬浮工具盘。它应该作为独立的小型管理面板展示，层级上和页面内工具盘分开，不要把 popup 画成网页里的普通浮层。

Popup 里需要展示：

- 顶部：PigeonDeck 名称、当前站点摘要。
- 状态：扩展运行中 / 已禁用、当前页面是否支持。
- 控件：全局禁用开关、当前站点禁用开关。
- 入口：禁用站点列表管理。
- 提示：file 权限未开启、PDF 页面不支持。

## 面板关系

请把面板做成有层级的，但不要堆很多窗口。

- 批注/编辑面板应贴近目标元素
- 设置面板应贴近工具盘
- 清空确认应贴近工具盘
- popup 应像浏览器扩展的小窗口，不要像独立应用

## 交互气质

请让动效慢而轻。

- 展开要柔和
- 收起要自然
- hover 要轻
- 激活态要克制
- 不能有夸张弹跳
- 不能有炫技式过渡

## 层次表达

请通过以下方式表达层次，但不要写得太重：

- 细边框
- 轻阴影
- 微弱明暗差
- 很轻的底色变化
- 很轻的状态色

不要让层级依赖厚边、重阴影、彩色大块或复杂背景。

## 主观感受

我希望看到的感觉是：

- 像一套很安静的工具
- 像宣纸上的轻轻标注
- 像懂分寸的人设计出来的产品
- 有高级感，但不是昂贵感
- 有温柔，但不是柔弱
- 有东方气质，但不是古风

## 你可以自由发挥的部分

以下部分你可以自由发挥，不要被我钉死：

- 具体配色的精确数值
- 字体名称
- 具体圆角大小
- 具体阴影参数
- 具体间距数值
- 具体图标库
- 具体布局微调

请优先自由发挥审美，但必须守住前面的气质、层级和禁区。

## 输出要求

请输出一个可以直接在浏览器里打开的单页 HTML 预览。  
这个页面可以是轻交互的，但不要做成复杂应用。  
我要的是一个能帮助我快速判断风格和层级的设计草稿。

如果你愿意，可以给页面加一点非常轻的状态切换，让我能看到：

- 悬浮球收起和展开
- 工具盘激活态
- 批注面板
- 高级样式区
- 区域批注
- 移动组件模式
- 设置面板
- 清空确认
- popup 片段

但请记住，重点仍然是审美和层级，不是复杂交互。
